GitHub+Hexo搭建个人博客

序言

欢迎光临我的博客网站!

偶然听说可以用GitHub+Hexo搭建个人博客，简直万分欣喜。博客是个好东西，即可以记录自己的日常心得，也可以归纳技术上的收获,闲来无事码上一篇，何乐而不为呢。
因此开篇我想说下GitHub+Hexo的搭建，供有相同爱好的朋友去开辟自己的空间。

Hexo简介

Hexo 是一个简单地、轻量地、基于Node的一个静态博客框架,可以方便的生成静态网页，引用作者的话说就是:快速、简单且功能强大的 Node.js 博客框架。

GitHub Pages

GitHub Pages 可以被认为是用户编写的、托管在github上的静态网页。由于它的空间免费稳定， 可以用于介绍托管在github上的Project或者搭建网站

1.环境搭建

说明本人用的Widnows环境，如果你用的LINUX或者MAC，自行下载对应的安装包即可。

安装Node.js

进入Node.js下载Node.js安装包，一路向下即可完成。
检查是否可用 在cmd 输入 npm 命令

如果提示无此命令，需自行配置一下nodejs在WINDOWS中的环境变量.

安装Git

进入Git下载界而,下载Git安装包,一路向下即可完成。

2.Hexo的安装配置

在任何一目录建立一文件夹比如说我在D盘根目录建立了 hexotyzs_blog 文件夹，用Git-CMD工具 进入该盘

至此hexo完成

启动hexo服务

这里建议用Git-Bash工具，后期也一直推荐这个工具。
进入hexotyzs_blog

打开浏览器访问本地页面 出现欢迎页面OK
按Ctrl+C 停止Server.

3.把Hexo部署至GitHub公网访问

注册GitHub账号

首先前往GitHub注册一个账号
然后创建一个仓库 建议Repository name 为 xxx用户名.github.io 创建成功之后你会发现Setting-Options-GitHub Pages 会给出你项目的访问链接

用Git-Bash工具 创建GitHub SSH-Key

配置GitHub仓库与本地Git关连

进入GitHub Setting-Options-Deploy keys 新增key把密钥中的代码贴过去OK.
进入hexotyzs_blog目录打开Git-Bash工具

上传代码公网访问

修改hexotyzs_blog目录下的_config.yml (此文件是hexo的配置文件，以后会经常用到)

保存修改进入hexotyzs_blog目录打开Git-Bash工具

部署代码OK 打开https://tyzs.github.io/访问公网地址。
至此GitHub+Hexo搭建个人博客完成。
